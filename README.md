# 文件共享服务

一个简单的Web文件共享服务，支持文件上传、下载和管理功能。

## 功能特性

- 🔐 用户认证: 用户名和密码登录保护
- 📁 浏览 `/share` 文件夹中的所有文件
- ⬆️ 上传文件到共享文件夹 (支持最大1GB)
- 🚀 智能分块上传 (大文件自动分块，提高稳定性)
- 📊 实时上传进度显示
- ⬇️ 下载共享文件夹中的文件
- 🗑️ 删除不需要的文件
- 📱 响应式设计，支持移动设备
- 🎨 现代化的用户界面
- 🖱️ 支持拖拽上传
- ⚡ 断点续传支持 (分块上传)

## 安装和运行

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 运行服务

```bash
python app.py
```

### 3. 访问服务

打开浏览器访问：http://localhost:5000

### 4. 用户登录

使用以下账户登录:
- **管理员**: 用户名 `root`, 密码 `123456`

登录后即可上传、下载和管理文件！

## 配置说明

- **共享文件夹**: `./share` (自动创建)
- **最大文件大小**: 1GB
- **分块上传阈值**: 50MB (超过此大小自动使用分块上传)
- **分块大小**: 1MB
- **服务端口**: 5000
- **访问地址**: http://localhost:5000 (局域网内其他设备可通过您的IP地址访问)

## 安全特性

- 🔐 **用户认证**: 基于会话的登录验证
- 🔑 **密码加密**: 使用Werkzeug密码哈希保护用户密码
- 🛡️ **访问控制**: 所有文件操作需要登录授权
- 🚫 **登录限制**: 失败次数限制和IP自动封禁
- 🔒 **会话管理**: 安全的会话密钥和超时机制
- 📝 **文件名检查**: 防止路径遍历攻击
- 💾 **文件大小限制**: 防止磁盘空间耗尽
- ⚠️ **错误处理**: 完善的错误处理和日志记录
- 🔍 **文件类型检测**: 支持的文件类型验证
- 📊 **安全日志**: 详细的登录和安全事件记录
- ⚙️ **管理界面**: 管理员可查看和管理安全状态

### 登录安全机制

#### 失败次数限制
- **最大失败次数**: 5次（可配置）
- **统计时间窗口**: 15分钟（可配置）
- **自动封禁**: 超过限制后自动封禁IP
- **封禁时长**: 30分钟（可配置）

#### IP封禁功能
- **自动封禁**: 登录失败次数超限时自动触发
- **手动解封**: 管理员可通过安全管理页面手动解封
- **状态监控**: 实时显示封禁状态和剩余时间
- **记录清除**: 管理员可清除IP的失败记录

#### 安全提示
- **剩余次数提醒**: 失败次数接近限制时显示警告
- **封禁通知**: 被封禁时显示剩余封禁时间
- **实时更新**: 登录页面实时显示安全状态

### 管理功能

#### 安全管理页面（仅限管理员）
- **访问路径**: `/admin/security`
- **功能包括**:
  - 查看当前安全配置参数
  - 监控被封禁的IP地址列表
  - 查看登录失败记录
  - 手动解封IP地址
  - 清除IP失败记录
  - 自动刷新状态（每30秒）



### 密码安全管理

#### 密码哈希加密
- **加密算法**: 使用Werkzeug的PBKDF2密码哈希
- **安全特性**: 自动加盐、防彩虹表攻击
- **存储方式**: 仅存储密码哈希值，不保存明文密码
- **验证过程**: 登录时使用安全的哈希比较

#### 密码管理工具

**方法一：使用密码生成脚本**
```bash
python generate_password_hash.py
```

**方法二：在Python中生成**
```python
from werkzeug.security import generate_password_hash
password_hash = generate_password_hash('your_password')
print(password_hash)
```

**方法三：使用内置工具函数**
```python
from app import generate_user_password_hash
password_hash = generate_user_password_hash('your_password')
```

#### 添加新用户
1. 使用上述任一方法生成密码哈希
2. 在 `app.py` 的 `USERS` 字典中添加:
```python
USERS = {
    'root': 'existing_hash',
    'new_user': 'generated_hash',  # 新用户
}
```

### 安全配置参数

可在 `app.py` 中修改以下安全参数：

```python
# 登录安全配置
MAX_LOGIN_ATTEMPTS = 5      # 最大失败次数
BAN_DURATION_MINUTES = 30   # 封禁时长（分钟）
ATTEMPT_WINDOW_MINUTES = 15 # 统计时间窗口（分钟）
```

**生产环境使用建议**:
1. 修改 `app.py` 中的 `SECRET_KEY`
2. 更改默认用户名和密码
3. 调整安全参数（失败次数限制和封禁时长）
4. 考虑集成更安全的认证系统
5. 启用HTTPS加密传输
6. 定期监控安全日志和封禁状态
7. 实施强密码策略和定期更换
8. 在受信任的网络环境中部署
